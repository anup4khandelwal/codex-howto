# 10 Playbooks

Playbooks turn Codex capabilities into repeatable workflows for specific jobs.

## Why this matters

Codex now spans local, cloud, IDE, GitHub, automation, and connector-backed work. The hard part is often choosing the right surface and giving it a task brief with enough boundaries to finish safely.

## Core concepts

- start from the workflow, not the feature name
- choose the surface based on state, review needs, and runtime
- write down the verification signal before work starts
- keep local and cloud handoffs explicit

## Common mistakes

- sending cloud tasks that depend on local uncommitted state
- using web search before defining the freshness question
- asking for visual UI work without screenshots or viewport checks
- treating a GitHub review as a complete fix plan without local verification
