# Codex Surfaces

Codex can be used from the CLI, IDE extension, web/cloud, GitHub, and connected workflow tools. Pick the surface based on where the work should happen.

## CLI

Use the CLI when the task is local, interactive, and benefits from close inspection of the working tree.

Good fits:

- small bugfixes
- focused refactors
- local test-driven changes
- repo orientation and explanation

## IDE extension

Use the IDE extension when you want Codex close to the editor and expect to review or apply changes while staying in your normal development loop.

Good fits:

- targeted edits around files already open
- follow-up questions while reviewing code
- starting a cloud task from editor context

## Web and cloud

Use web/cloud when Codex should work in the background, especially when tasks can run in parallel or do not require your local uncommitted state.

Good fits:

- issue-to-PR work
- independent bugfixes
- repo-wide investigation
- tasks that benefit from isolated environments

## GitHub and workflow integrations

Use GitHub, Slack, or Linear when the source of truth already lives in a PR, issue, review thread, or team conversation.

Always restate:

- the repository
- the expected outcome
- the verification signal
- the boundary of what should not change
