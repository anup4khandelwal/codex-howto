# Small Feature

Use this skill for a bounded implementation task that should fit in a small change set.

## Workflow

1. Orient to the relevant part of the repo before editing.
2. Find one existing pattern to mirror.
3. Write the smallest change that satisfies the request.
4. Add or extend tests for the changed behavior.
5. Verify the result and summarize the diff clearly.

## Output contract

Include:

- the files changed
- the implementation approach
- the verification run
- any follow-up work intentionally left out of scope

## Avoid

- architecture churn for a small request
- broad renames or unrelated cleanup
- shipping without a clear verification result
